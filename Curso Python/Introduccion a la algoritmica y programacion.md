# Nociones Previas
 
 ## Dato
 Un dato es un hecho o cifra cruda que, por sí solo, carece de significado. Es el material básico que puede ser recopilado, observado o medido. Ejemplo: "10,000" como dato aislado no tiene contexto.
 
 ## Información
 La información surge al organizar, procesar e interpretar los datos para darles sentido. Es la conexión entre los datos que permite comprender algo. Ejemplo: "10,000 pasos al día son recomendados para mantener una vida activa y saludable."
 
 ## Conocimiento
 El conocimiento es la aplicación y comprensión de la información en base a experiencias, aprendizaje y contexto. Es lo que permite tomar decisiones, resolver problemas y generar ideas. Ejemplo: "Caminar 10,000 pasos al día contribuye a mejorar la salud cardiovascular según estudios científicos."

 # Información y sus cualidades
 
 ## Precisión
 La precisión se refiere al porcentaje de información correcta dentro del total de datos disponibles en un sistema. Por ejemplo, en un cálculo de costos, se puede tolerar un margen de error mínimo. Sin embargo, en un análisis estadístico, la información debe ser exacta para mantener su validez y evitar alterar significativamente los resultados.
 
 ## Integridad
 La integridad implica que la información sea coherente y refleje fielmente el mundo real. Debe respetar las reglas semánticas y proporcionar una visión precisa de la realidad para ser útil.
 
 ## Significado
 El significado está relacionado con el valor semántico de la información. La información debe ser comprensible y relevante para el usuario. Proveer demasiados datos sin contexto o utilidad puede resultar en una sobrecarga informativa que pierde su significado esencial.
 
 ## Oportunidad
 La oportunidad indica que la información debe estar disponible en el momento en que se necesita. Su relevancia disminuye con el tiempo, y, pasado cierto punto, puede incluso perder su utilidad por completo.
 
 ## Plenitud
 La plenitud se refiere a la necesidad de que la información sea completa para cumplir su propósito. Por ejemplo, en la toma de decisiones empresariales, los datos deben incluir todos los elementos informativos necesarios para apoyar una decisión informada. La información incompleta puede generar errores significativos.

 # Hardware y Software
 
 
 ## Hardware
 
 
 El hardware se refiere a los componentes físicos y tangibles de un sistema informático. Es todo aquello que puedes tocar y ver. Algunos ejemplos incluyen:
 ### ¿Por qué la importancia del hardware?}
 
 - **Unidad Central de Procesamiento (CPU):** El "cerebro" de la computadora que ejecuta instrucciones.
 - **RAM (Memoria de Acceso Aleatorio):** Memoria temporal que almacena datos mientras se usan.
 - **Disco duro o SSD:** Almacenan datos de forma permanente.
 - **Periféricos:** Dispositivos externos como el teclado, el ratón, la impresora y el monitor.
 - **Placa madre:** El componente principal que conecta y comunica todos los elementos del hardware.
 
 ## Software
 El software representa los programas y sistemas operativos que permiten que el hardware funcione y realice tareas específicas. Algunos tipos de software incluyen:
 
 ### ¿Por qué la importancia del software?
 - **Sistema Operativo:** Como Windows, macOS, Linux y Android; gestiona todos los recursos del hardware y permite interactuar con el sistema.
 - **Aplicaciones:** Programas diseñados para realizar tareas específicas, como Microsoft Word, Excel, navegadores web (Chrome, Firefox) o aplicaciones de redes sociales como Facebook.
 - **Software de utilidad:** Herramientas para mantenimiento y seguridad, como antivirus, programas para hacer copias de seguridad, y optimizadores de rendimiento.
 - **Drivers:** Programas que permiten la comunicación entre el sistema operativo y los dispositivos de hardware.
 
 Hardware y software trabajan en conjunto para proporcionar un sistema informático funcional y eficiente. ¡Todo está conectado para facilitar tareas desde las más simples hasta las más complejas!

 # <span style="color:#00bfbf;">**¿Qué es un Algoritmo?**</span>
 
 > Un **algoritmo** es un conjunto de pasos con instrucciones claras, finitas y organizadas que se sigue para resolver un problema específico o realizar una tarea.  
 >  
 > Es como una **receta** que detalla las acciones necesarias para alcanzar un objetivo, garantizando un resultado óptimo y eficiente.
 
 ---
 
 ## <span style="color:#ff8c00;">¿Y en programación?</span>
 
 En **programación**, un algoritmo supone el paso previo a ponerse a escribir el código.  
 1. **Primero** se debe pensar cómo se afrontará el problema (definir la solución al problema).  
 2. **Después**, se traduce esa solución a un lenguaje de programación.  
 
 Este método, que consiste en darle a la máquina instrucciones paso a paso para luego traducirlas a un lenguaje de programación, se conoce como **algoritmo**.  
 
 > Así nacen los llamados **diagramas de flujo** o **pseudocódigo**, donde los algoritmos son concebidos y, posteriormente, **codificados** en un lenguaje de programación para poder ser **ejecutados** en un ordenador.
 
 ---
 
 ## <span style="color:#7cfc00;">Ejemplo de diagrama de flujo: La lámpara no funciona</span>
 
 
 1. **¿Está enchufada?**  
    - **No** → Enchufarla  
    - **Sí** → Pasar a comprobar el foco  
 
 2. **¿Foco quemado?**  
    - **No** → Si sigue sin funcionar, revisar otras causas  
    - **Sí** → Reemplazar el foco  
 
 3. **Si nada funciona** → Comprar una nueva lámpara  
 
 ---
 
 ### <span style="color:#b8860b;">Conclusión</span>
 
 Los algoritmos nos ayudan a **organizar** nuestras ideas y **pasos** para resolver problemas de forma **eficiente**.  
 En programación, son la base para **traducir** nuestras soluciones a un lenguaje que la **computadora** pueda entender.

 # <span style="color:#00bfbf;">**Partes de un algoritmo**</span>
 
 Las tres partes de un algoritmo son:
 
 1. ### **Input (Entrada)**
    - Información que damos al algoritmo y con la que va a trabajar para ofrecer la solución deseada.
 
 2. ### **Proceso**
    - Conjunto de pasos para que, a partir de los datos de entrada, se realicen operaciones lógicas o aritméticas.
 
 3. ### **Output (Salida)**
    - Resultados que se obtienen a partir de la transformación de los valores de entrada en datos útiles.
 
 ---
 
 # <span style="color:#00bfbf;">**Características de los algoritmos**</span>
 
 1. **Precisos**  
    <span style="color:green">Objetivos, sin ambigüedad.</span>
 
 2. **Ordenados**  
    <span style="color:green">Presentan una secuencia clara y precisa para llegar a la solución.</span>
 
 3. **Finitos**  
    <span style="color:green">Contienen un número determinado de pasos.</span>
 
 4. **Concretos**  
    <span style="color:green">Ofrecen una solución determinada para un problema planteado.</span>
 
 5. **Definidos**  
    <span style="color:green">El mismo algoritmo, con la misma entrada, debe ofrecer la misma salida.</span>

 # <span style="color:#00bfbf;">**Programa**</span>
 
 El término *programa* tiene diferentes acepciones, pero desde la perspectiva de la informática podemos definirlo como:
 
 > **"Codificación de un algoritmo en algún lenguaje de computadora"**
 
 ---
 
 ## <span style="color:#ff8c00;">Algoritmo y programa</span>
 
 - Un **algoritmo** describe los pasos lógicos para resolver un problema.
 - Un **programa** es la traducción de ese algoritmo a un lenguaje de programación.
 
 > Es importante reiterar que primero se elabora el **algoritmo** y luego se escribe el **programa**.  
 > Muchas personas obvian la etapa del algoritmo y pretenden afrontar un problema empezando directamente por la programación, lo cual **no** es conveniente.
 
 ---
 
 ## <span style="color:#ff8c00;">Ejemplo de relación entre problema, algoritmo y programa</span>
 
 ```mermaid
 flowchart LR
     A(Problema) --> B[Lógica de solución <br><span style="font-size:0.8em">(Análisis y diseño)</span>]
     B --> C[Algoritmo <br>(Pseudocódigo)]
     C --> D[Programa <br>(Código en un lenguaje)]
 
 ```

 ## Lenguaje de Programacion : *Léxico + Sintaxis + Semántica*
 
 >Un lenguaje de programación es un conjunto de símbolos(con un cierto significado) y ciertas reglas de sintaxis que permiten escribir programas para el computador(comunicación con el computador). En términos mas formales, un lenguaje de programación esta formado por un léxico, una sintaxis y una semántica asociada.
 
 * **Léxico** : Conjunto de símbolos, códigos o palabras reservadas que utiliza el lenguaje
 * **Semantica** : Conjunto de reglas necesarias para construir expresiones o sentencias correctas que constituirán un programa, utilizando el léxico del lenguaje.
 * **Semántica** : Sentido o significado que se le da a las construcciones que constituyen el programa.
 
 ## Pseudocódigo
 >Una herramienta utilizada para diseñar y planificar algoritmos de manera sencilla y clara, antes de programarlos en un lenguaje especifico. Sus principales caracteristicas son:
 >* El lexico del pseudocodigo puede estar formado por simbolos y codigos definidos por el usuario, esto se debe a que no depende de un lenguaje específico
 >* No sigue las reglas estrictas de sintaxis de un lenguaje de programacion
 >* Ayuda a visualizar la lógica de un programa de forma estructurada.
 
 **Ejemplos**
 ```
     Inicio
         Leer número
         Si número > 0 Entonces
             Escribir "El número es positivo"
         Sino
             Escribir "El número es negativo"
         Fin_Si
     Fin
 ```

 # 🖥️ Lenguaje de Programación: **Léxico + Sintaxis + Semántica**

 📌 Conjunto de **reglas necesarias** para construir las expresiones o sentencias correctas que formarán un programa, utilizando el léxico del lenguaje.  
 
 ### 🔸 Semántica  
 📌 **Sentido o significado** que se le da a las construcciones que constituyen el programa.

 # 📌 **PSEUDOCÓDIGO**
 
 El **pseudocódigo** es una herramienta utilizada para diseñar y planificar algoritmos de manera sencilla y clara, antes de programarlos en un lenguaje específico. 
 
 ## ✨ **Principales características**:
 - 📌 El léxico del pseudocódigo puede estar formado por símbolos y códigos definidos por el usuario, evitando depender de un lenguaje específico.
 - 📌 No sigue las reglas estrictas de sintaxis de un lenguaje de programación.
 - 📌 Ayuda a visualizar la lógica de un programa de forma estructurada.
 
 ---

 ```plaintext
 Inicio
     Leer número
     Si número > 0 Entonces
         Escribir "El número es positivo"
     Sino
         Escribir "El número es negativo"
     Fin_Si
 Fin
 
 
 ```
